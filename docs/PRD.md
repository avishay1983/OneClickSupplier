# מסמך דרישות מוצר (PRD)
## מערכת "ספק בקליק" - הקמת וניהול ספקים

**גרסה:** 1.0  
**תאריך:** דצמבר 2024  
**ארגון:** ביטוח ישיר

---

## 1. סקירה כללית

### 1.1 מטרת המערכת
מערכת "ספק בקליק" היא פלטפורמה דיגיטלית מקיפה לניהול תהליך הקמת ספקים חדשים בארגון, מרגע יצירת הבקשה ועד לאישור הסופי וניהול שוטף של הספק.

### 1.2 קהל יעד
- **מנהלי רכש** - יצירת בקשות, מעקב ואישור ספקים
- **סמנכ"ל** - אישור וחתימה על הסכמים (במקרים הנדרשים)
- **מטפלים** - ניהול יומיומי של בקשות ספקים
- **ספקים** - מילוי טפסים, העלאת מסמכים, הגשת הצעות מחיר וקבלות
- **מנהלי מערכת** - ניהול משתמשים והגדרות

## 2. תהליכים עסקיים

### 2.1 תהליך הקמת ספק חדש

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   יצירת בקשה    │────▶│  הספק ממלא טופס │────▶│   בקרה ראשונה   │
│   ע"י מטפל      │     │   + מסמכים      │     │                 │
└─────────────────┘     └─────────────────┘     └────────┬────────┘
                                                         │
         ┌───────────────────────────────────────────────┘
         ▼
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  אישור מנהלים  │────▶│   חתימות על    │────▶│    ספק אושר     │
│                │     │    חוזה        │     │                 │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

#### שלבי התהליך:

**שלב 1: יצירת בקשה**
- מטפל מתחבר למערכת ויוצר בקשה חדשה עם פרטי הספק הבסיסיים
- בשלב זה נקבע האם הספק דורש אישור סמנכ"ל (בהתאם לסוג העסקה או הסכום)
- המערכת שולחת קישור מאובטח לספק במייל
- תוקף הקישור מוגבל (ברירת מחדל: 7 ימים)

**שלב 2: מילוי טופס הספק**
הספק מקבל קישור ייחודי ומבצע:
- אימות זהות (קוד חד-פעמי למייל)
- העלאת מסמכים נדרשים:
  - אישור ניהול ספרים
  - אישור ניכוי מס במקור
  - צילום המחאה / אישור בנק
  - צילום חשבונית
- מילוי פרטים:
  - פרטי חברה (ח.פ, טלפון, כתובת)
  - אנשי קשר (הנהלת חשבונות, מכירות)
  - פרטי בנק
  - אמצעי תשלום מועדף
- העלאת הצעת מחיר חתומה (אופציונלי)

**שלב 3: עיבוד אוטומטי**
- זיהוי אוטומטי של סוג המסמכים שהועלו
- חילוץ אוטומטי של פרטי בנק מהמסמכים
- השוואה בין נתונים מחולצים לנתונים שהספק הזין
- התראה למטפל במקרה של אי-התאמות

**שלב 4: בקרה ראשונה**
- מטפל סוקר את הטופס והמסמכים
- מאשר או דוחה את הבקשה
- במקרה של אישור - הבקשה עוברת לשלב האישורים

**שלב 5: אישור מנהלים**

> **חשוב:** רמת האישור הנדרשת נקבעת בעת יצירת הבקשה

| סוג בקשה | אישור נדרש |
|----------|------------|
| בקשה רגילה | אישור מנהל רכש בלבד |
| בקשה הדורשת סמנכ"ל | אישור מנהל רכש + אישור סמנכ"ל |

- כל מאשר מקבל מייל עם קישור לאישור
- המאשר יכול לאשר או לדחות (עם סיבה)

**שלב 6: חתימה על הסכם**
- חתימה דיגיטלית בהתאם לרמת האישור שנקבעה:
  - בקשה רגילה: חתימת מנהל רכש בלבד
  - בקשה עם סמנכ"ל: חתימת סמנכ"ל + חתימת מנהל רכש
- שמירת ההסכם החתום
- שליחת הודעה לספק על אישור סופי

### 2.2 תהליך הצעות מחיר

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  שליחת בקשה    │────▶│  הספק מעלה      │────▶│   אישור מטפל   │
│  להצעת מחיר    │     │  הצעת מחיר     │     │                 │
└─────────────────┘     └─────────────────┘     └────────┬────────┘
                                                         │
         ┌───────────────────────────────────────────────┘
         ▼
┌─────────────────┐     ┌─────────────────┐
│  אישור סמנכ"ל  │────▶│  אישור מנהל    │
│  (אם נדרש)     │     │    רכש         │
└─────────────────┘     └─────────────────┘
```

#### תיאור התהליך:
- מטפל שולח לספק קישור להגשת הצעת מחיר
- הספק מעלה את ההצעה עם ציון סכום ותיאור
- ההצעה עוברת תהליך אישור רב-שלבי (בהתאם לסכום/סוג)
- בסיום - חתימה דיגיטלית על ההצעה

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐

│   לקבלות       │     │    קבלות       │     │                 │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

#### תיאור התהליך:
- מטפל שולח לספק קישור להעלאת קבלות
- הספק מעלה קבלות עם ציון סכום, תאריך ותיאור
- מטפל סוקר ומאשר/דוחה כל קבלה
- במקרה דחייה - נשלחת סיבה לספק


- לאחר אישור - המשתמש יכול להתחיל לעבוד במערכת

## 3. ממשקי המערכת

### 3.1 לוח בקרה (Dashboard)

**מי משתמש:** מטפלים ומנהלים

#### תצוגת מטפל:
- רשימת הבקשות שהמטפל מטפל בהן
- יצירת בקשה חדשה (יחידה או יבוא מאקסל)
- סטטוס כל בקשה
- פעולות: צפייה במסמכים, עריכה, שליחה מחדש, אישור/דחייה

#### תצוגת מנהל:
- רשימת בקשות הממתינות לחתימה
- גישה מהירה לחתימה דיגיטלית
- צפייה בכל הבקשות (למנהלי מערכת)

---

### 3.2 טופס הספק

**מי משתמש:** ספקים

- טופס אינטרנטי נגיש דרך קישור מאובטח
- שני שלבים: העלאת מסמכים → מילוי פרטים
- זיהוי ומילוי אוטומטי של שדות
- אימות זהות בקוד חד-פעמי

---

### 3.3 מערכת ניהול ספקים (CRM)

**מי משתמש:** מטפלים ומנהלים

- רשימת כל הספקים המאושרים
- חיפוש וסינון לפי: סטטוס, שם, סוג ספק
- ניהול סטטוס ספק (פעיל, מושהה, סגור, VIP)
- צפייה בהיסטוריית שינויים
- דירוג ספקים (1-5 כוכבים)
- ניהול הצעות מחיר וקבלות

---

### 3.4 עמוד אישור הצעת מחיר

**מי משתמש:** מנהלים

- צפייה בהצעת המחיר
- אפשרות לאשר או לדחות
- חתימה דיגיטלית על ההצעה

---

### 3.5 עמוד העלאת קבלות

**מי משתמש:** ספקים

- העלאת קבלה חדשה
- ציון פרטי הקבלה (סכום, תאריך, תיאור)
- צפייה בקבלות קודמות וסטטוסן

---

### 3.6 הגדרות מערכת

**מי משתמש:** מנהלי מערכת בלבד

- הגדרת פרטי סמנכ"ל (שם, מייל)
- הגדרת פרטי מנהל רכש (שם, מייל)
- הגדרת פרטי מנהל מערכת (שם, מייל)

## 4. סטטוסים

### 4.1 סטטוסי בקשת ספק

| סטטוס | משמעות |
|-------|--------|
| ממתין | נוצרה בקשה, טרם נשלח קישור |
| ממתין לספק | נשלח קישור לספק |
| הוגש | הספק שלח את הטופס |
| בקרה ראשונה | עובר סקירה ראשונית |
| אושר | כל האישורים והחתימות הושלמו |
| נשלח מחדש | הקישור נשלח מחדש לספק |
| נדחה | הבקשה נדחתה |

### 4.2 סטטוסי ספק ב-CRM

| סטטוס | משמעות |
|-------|--------|
| פעיל | ספק פעיל ועובד |
| מושהה | פעילות הספק הושהתה זמנית |
| סגור | הספק כבר לא עובד עם הארגון |
| VIP | ספק עם סטטוס מועדף |
| אושר ביטחון | ספק שעבר אישור ביטחוני |

### 4.3 סטטוסי הצעת מחיר וקבלה

| סטטוס | משמעות |
|-------|--------|
| ממתין | ממתין לסקירה |
| אושר | אושר |
| נדחה | נדחה (עם סיבה) |

## 7. יכולות חכמות

- **זיהוי מסמכים:** המערכת מזהה אוטומטית את סוג המסמך שהועלה
- **חילוץ פרטי בנק:** חילוץ אוטומטי של פרטי בנק מהמחאה/אישור בנק
- **מילוי אוטומטי:** פרטי חברה ממולאים אוטומטית מהמסמכים
- **התראות על אי-התאמות:** המערכת מתריעה כאשר יש פער בין הנתונים
